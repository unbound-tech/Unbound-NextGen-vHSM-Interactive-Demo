Client CLI ?

Note: The UKC command line interface described here is only available when the vHSM demo is run locally.

The UKC client command line interface (CLI) lets you interact with UKC. To open the CLI, you need to open a command shell on the UKC client (ukc-client).

For example, run this command to open a Bash shell: docker exec -w /pipelines -it ukc-client /bin/bash

After opening the shell, you can run CLI commands. The following command lists the keys in UKC: ucl list

Demos are provided in scripts as follows:
  • /pipelines/01-sign_docker_image.sh - Sign a Docker image.
  • /pipelines/02-verify_docker_image.sh - Verify the signed image.
  • /pipelines/03-sign_jar_file.sh - Sign a Java jar file.
  • /pipelines/04-verify_jar_file.sh - Verify the signed jar file.
  • /pipelines/05-sign_rpm_file.sh - Sign an RPM file.
  • /pipelines/06-verify_rpm_file.sh - Verify the signed RPM file.
  • /pipelines/07-kmip.sh - Run a demo of a KMIP client.

You may want to open the System Logs or the UKC Admin portal to see how your CLI commands affect UKC.

See the CLI section of the UKC Interface Guide for more information.
